Rule 1. We don't write code until we understand the problem.

Steps to design a system
1. What are the responsibilities?
2. Turn responsibilities into modules
3. Design the contracts

Engineering Principles
1. Design responsibilities before designing functions.
2. One responsibility. One owner.
3. Who needs this return value?
4. Data should flow downward
5. Choose the right data structure first.
6. High-level modules should coordinate. Low-level modules should specialize.
